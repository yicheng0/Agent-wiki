## 一、Solopreneur 的老问题：信息差不是找得到，是每天都能稳定找得到

我是AI Solopreneur，每天比别人早一步掌握 AI Valley 的动向，是我做内容最重要的输入。

过去一年我的习惯是早起刷一个小时 X。刷完整个人都是兴奋的，因为总能发现几个别人还没注意到的点。但这个习惯有两个问题。

**一是不稳定**。状态好的时候刷得快，状态差的时候刷到一半走神，就把重要推文漏了。 **二是不可复制**。每一次刷到好内容的逻辑都在我脑子里，关掉 X 标签页，一切归零。明天早上还要重新来一遍。

信息差这件事的本质不是"找得到"，是"每天都能稳定找得到"。单次找到不值钱，稳定持续才值钱。

我之前也试过用各种 AI 产品辅助筛选，但是每次都一样：聊一次出一份报告很惊艳，关掉标签页，明天起床又从零开始描述一次需求。

这些高质量对话在我这里没有沉淀，都是一次性劳动。

## 五、两次现场 Tool Making：从 v2.1.0 一路迭代到 v2.3.0

第一天跑出来的 dashboard 已经很漂亮了（v2.1.0 自带 Gmail 邮件发送）。但用了一天我发现两个小痒点，随手跟 Agent 提了一下，它几十分钟里连着升了两次。

第一次升级：原推文直链

我说：**"精选信息差每一条的标题，我希望能一键跳回原推。"**

它答："好，我在每条后面加一个「→原文」按钮，点了直接跳 X 原页面。"

30 分钟后，Agent 升到 v2.2.0。精选信息差 10 条每条都挂上了原推链接。从这一刻开始，我在 Notion 里读摘要、一键跳 X 看原文，形成了完整闭环。

第二次升级：og:image 预览图

第二天我又提一个需求：**"Top 3 互动那里，能不能给每条推文配上原文的截图预览？光看标题不够，我还是得点进去才知道内容长什么样。"**

它想了几秒，然后告诉我它打算怎么做：用 **Twitterbot UA** 的身份去抓推文的 **og:image**，这样不用申请 X 的图片 API 权限，就能把推文卡片的官方预览图拉回来。

我愣了一下。**Twitterbot UA 这个 trick 我自己都不知道。**

20 分钟后，Agent 升到 v2.3.0。新 dashboard 里 Top 3 每条都带上了完整的推文预览截图，清晰到能直接读出推文正文。顶部多了一个"3/3 预览图成功"的数据卡片，它自己加上去的，我们看看实际效果。

![图像](https://pbs.twimg.com/media/HGgDwsdaYAAzy1X?format=jpg&name=large)

为什么这两次升级很重要？

**我缺一个能力，Agent 不是回我"这个我做不了"，也不是让我去提 feature request 排队等平台更新。它现场想一个方案自己造出来，几十分钟就用上了。**

v2.1 → v2.2 是它给自己加了一个按钮渲染能力，v2.2 → v2.3 是它给自己开发了一个 og:image 抓取能力。两次都不是 CREAO 平台预设里原生支持的功能，是 Agent 用自己手里的代码执行 + 网页抓取 + 文件读写这几块原子能力，组合出来的一个**专门为我这个场景服务的小工具**。

这就是 Tool Maker 那一层意思。

如果让我用一句话总结这个瞬间：**它不是一个更会用工具的 Agent，是一个会给自己造工具的同事。**

**这是 AI 增强个人能力的正确姿势：不是外包你的大脑，是外包你已经想清楚的部分。**
